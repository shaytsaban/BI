שיעור 5: סקרייפינג נתונים, API ובניית אפליקציות Web

מרצה: ד״ר שי צבאן

בשיעור זה נלמד לקחת את הידע התיאורטי שלנו ולהפוך אותו למוצר טכנולוגי חי ונושם. ניקח נתונים מהעולם האמיתי, נמשוך אותם באוטומציה, ונציג אותם בצורה ויזואלית מרהיבה.

איפה אנחנו נמצאים? 🗺️

הפער בין נתונים מתים לנתונים חיים

עד עכשיו בקורס, בנינו מודלים, דיברנו על מדדים פיננסיים, ולמדנו להציג נתונים במערכות כמו Tableau. אבל הייתה לנו מגבלה אחת גדולה:

הסתמכנו על קובצי Excel סטטיים.
הורדנו קובץ, טענו אותו, וניתחנו. מה קורה מחר כשהבורסה נפתחת והנתונים משתנים? האם נוריד קובץ מחדש כל בוקר?

היום אנחנו עוברים מ"אנליסטים שעובדים קשה" ל"מפתחי מערכות מידע שעובדים חכם". אנחנו נבנה מנגנון שעושה את זה עבורנו.

  • אוטומציה מלאה: הסקריפט עובד בשבילנו 24/7.
  • חיבור לעולם: שאיבת נתוני מניות בזמן אמת.
  • ממשק משתמש: יצירת "פרונט-אנד" (Frontend) שמאפשר לכל משתמש פשוט ללחוץ על כפתור ולקבל תובנות.

הכירו את Google Apps Script 🚀

המנוע השקט שמאחורי הקלעים (Back-end)

כדי לבצע אוטומציות ברשת, אנחנו צריכים "שרת". מחשב שתמיד דולק ומחכה לפקודות שלנו. גוגל מספקת לנו אחד כזה בחינם לחלוטין!

מה זה בכלל GAS?

Google Apps Script היא שפת תכנות (מבוססת JavaScript, למעשה זו אותה שפה) שרצה ישירות על השרתים של גוגל בענן. היא מאפשרת לנו לגשת לג'ימייל, לגוגל שיטס, וחשוב מכל - לצאת לאינטרנט החיצוני (למשל לאתר גוגל פיננס) ולשאוב ממנו מידע!

איך ניגשים לזה? פשוט מאוד:

  • פותחים גוגל דרייב.
  • יוצרים קובץ Google Sheets חדש.
  • בתפריט העליון לוחצים על: הרחבות (Extensions) ➔ Apps Script.
  • בום 💥 - נפתח לכם עורך קוד ריק. זה המטבח שלכם.

איך כותבים את הקוד? AI כשותף 🤖

אין לכם רקע בתכנות? אין שום בעיה.

בעידן של היום, מערכות המידע לא דורשות מכם לשנן סינטקס בעל פה. הן דורשות מכם לדעת להנדס את הבקשה (Prompt Engineering).

אתם יכולים לגשת למודלי שפה (כמו שאנחנו עושים ממש עכשיו) ולבקש מהם לכתוב עבורכם את הסקריפט.

נוסחת הפרומפט המושלם:
1. תפקיד: "אתה מפתח מומחה ב-Google Apps Script."
2. משימה: "כתוב לי פונקציה שמקבלת סימול של מניה..."
3. מקור נתונים: "...ושואבת את המחיר שלה מ-Google Finance."
4. פלט: "...תחזיר את הנתון כמספר עשרוני."

זמן התנסות: נסחו פרומפט

איך הייתם מבקשים מ-AI לכתוב סקריפט ששואב את שער הדולר העדכני מול השקל?

סקרייפינג ו-API: מאיפה באים הנתונים? 🕸️

איך המחשב קורא עיתון כלכלי

מה זה סקרייפינג (Scraping)?

קצירת נתונים. במקום שאדם ייכנס לאתר, יסמן את המחיר של מניית טסלה, יעשה "העתק" וידביק באקסל, אנחנו כותבים סקריפט שקורא את קוד ה-HTML של האתר, מחפש את המילה "מחיר", ושולף רק את המספר שלידה.

מה זה API?

(Application Programming Interface). זה המלצר במסעדה. במקום שהסקריפט שלנו (הלקוח) יפרוץ למטבח (האתר הפיננסי) ויחטט בסירים (ב-HTML) כדי למצוא נתונים, ה-API הוא תפריט מסודר. האתר אומר לנו: "רוצה את מחיר המניה? פשוט תשלח לי בקשה לכתובת הזו, ואני אחזיר לך קובץ נתונים נקי ומוכן".

// דוגמה לנתון שמגיע מ-API (פורמט JSON)

{ "ticker": "TSLA", "price": 185.32, "currency": "USD", "time": "2026-04-28 10:00:00" }

הסקריפט שלנו לוקח את הנתון הנקי הזה, ומעביר אותו אלינו.

חיבור ה-HTML ל-GAS 🔗

הלקוח פוגש את המטבח

זוכרים את המהלך שעשינו? יש לנו את אתר הקורס הראשי. שם יש לנו קובץ HTML. זה ה"פרונט-אנד" (Frontend) שלנו. שם הסטודנט, או מנהל הכספים, רואה כפתורים יפים, שדות חיפוש וגרפים צבעוניים. ויש לנו את קובץ ה- .gs (ב-GAS) שהוא ה"בק-אנד" (Backend) שעושה את העבודה הקשה.

איך הם מדברים אחד עם השני?

HTML Button Click ➔ google.script.run ➔ GAS Script

באמצעות פקודת קסם ב-JavaScript בצד ה-HTML, האתר פונה לשרת של גוגל, מפעיל את הפונקציה שכתבנו, ממתין לתשובה, וכשהמחיר חוזר - הוא מציג אותו למשתמש!

📌 הערה חשובה לסטודנטים: כל הקודים המלאים קיימים באתר הקורס הראשי! גם ה-HTML וגם ה-GS. אתם מוזמנים להוריד אותם, לפתוח את ה-HTML בדפדפן, ולהתחיל לשחק עם הקוד.

בואו נבדוק מה הבנו 🤔

שאלות חשיבה למערכות מידע
1. היכן רץ (מתבצע) קוד ה-Google Apps Script ששואב את הנתונים הפיננסיים?
א. על המחשב הנייד שלי (בדפדפן שמציג את ה-HTML).
ב. בשרתי GitHub Pages בהם פרסמנו את אתר הפרונט-אנד.
ג. בשרתים של גוגל בענן.
ד. בתוך מסד הנתונים של Tableau.
חשבו מי מספק את שירות ה-GAS ולמה קוראים לזה "צד שרת" (Backend).
2. שאלה מתחכמת: האם אפשר לבנות תהליך של סקרייפינג ואיסוף נתוני מניות באופן יומיומי ללא קובץ HTML (פרונט-אנד) כלל?
א. לא, ה-HTML הוא זה שנותן את פקודת ההפעלה (ה"טריגר").
ב. כן בהחלט. אפשר להגדיר ב-GAS מנגנון אוטומטי (Time-driven trigger) שירוץ לבד בכל בוקר.
ג. כן, אבל הנתונים יימחקו ברגע שנסגור את הדפדפן.
ד. לא, ה-HTML אחראי על שמירת הנתונים בבסיס הנתונים.
מה תפקיד הפרונט-אנד לעומת הבק-אנד? האם ה"מטבח" יכול לבשל גם אם אין לקוחות במסעדה?

איך מפרסמים את הקסם הזה? 🌐

שלבי Deployement (פריסה)

כדי שה-HTML שלנו יוכל לדבר עם ה-GAS, אנחנו חייבים להפוך את הסקריפט שלנו לנגיש ברשת, כ"אפליקציית רשת" (Web App).

  1. במסך של ה-Apps Script, לחצו בצד ימין למעלה על Deploy (פריסה).
  2. בחרו ב- New deployment.
  3. לחצו על גלגל השיניים ובחרו סוג: Web app.
  4. הגדירו גישה: "Anyone" (כדי שגם המשתמשים שיפתחו את ה-HTML יוכלו למשוך נתונים ללא הרשאות גוגל מיוחדות).
  5. העתיקו את ה-URL (כתובת ה-API שלכם!) והדביקו אותו בקוד ה-HTML במקום המסומן.

את דף ה-HTML עצמו (הפרונט-אנד), כפי שלמדנו בעבר, אתם מפרסמים בחינם ב- GitHub Pages. עכשיו יש לכם מערכת מלאה באוויר, מבוססת ענן, מנתחת נתונים בזמן אמת, ומוכנה להצגה בפני הנהלת החברה.
